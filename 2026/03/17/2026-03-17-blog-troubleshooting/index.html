Hexo 博客部署踩坑全记录

今天花了一整天时间，把我的 Hexo 博客从配置、改名到部署全部理顺了。
踩了无数坑，也终于找到了最稳定、最不折腾的部署方式，记录一下，给自己留个备忘，也帮同样是新手的朋友避坑。

一、今天遇到的问题

  1. SSH 权限报错
    git@github.com: Permission denied (publickey)
    公钥配置了，但 ssh-agent 启动失败，ssh-add 一直报错。

  2. HTTPS 部署网络不稳定
    Recv failure: Connection was reset
    国内网络访问 GitHub 经常抽风，部署一半就断。

  3. Git 分支不匹配
    error: src refspec main does not match any
    本地是 master，远程是 main，直接推 main 会失败。

  4. 批处理 .bat 脚本运行失败
    环境变量、路径、编码问题，导致双击没反应。

二、一步步解决过程

1. 放弃 SSH，改用本地 Git 强制推送

SSH 对 Windows 新手太不友好，直接改用手动推送：
hexo g 生成网站
进入 public 手动提交
强制推送到远程

正确命令：
git add .
git commit -m “更新博客”
git push -f origin master:main

2. 修复分支问题

本地默认分支：master
GitHub 远程分支：main
必须使用 master:main 格式才能推送成功。

3. 修复一键部署 .bat 脚本

解决了：
编码乱码
环境变量找不到
路径错误
最终写出真正可用的双击自动部署脚本。

三、最终成果

✅ 博客名称成功改为：虾滑的校园技术小站
✅ 描述改为：一只爱写代码、爱记录生活的虾滑
✅ 部署彻底稳定，再也不会传不上去
✅ 拥有了双击即用的一键部署脚本
✅ 网站正常更新，GitHub Pages 同步成功

四、新手避坑建议

  1. 别死磕 SSH，Windows 下手动 Git 更稳定
  2. hexo g 生成成功，网站就不会丢
  3. 分支不对就用 master:main 强制推送
  4. 重复操作写成脚本，能省超多时间

五、小结

折腾了一天，虽然过程很崩溃，但最后全部搞定的感觉真的超爽！
以后终于可以安心写文章，不用再被部署折磨啦～
